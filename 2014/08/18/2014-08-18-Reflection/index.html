Reflection--Lessons From A Project

Posted on 2014-08-18   |  
  • Recently, I have finished developing a new module, called accesslog, of which the main function is to read the log infos from database and to display these infos.

    A brief view of accessLog

  • It takes me almost the whole week to design and implement all the functions of this module, including drawing charts and tables, fuzzy and advanced search, as well as html events. This project, accessLog, differs from my school assignments dramatically, though which may be much harder than accessLog. The main difference between accessLog and school assignments is that there are so much more aspects I have to cover in accessLog. From the abstract design to the specific functions, surely, these aspects are not be as hard as implementing a PV-Lock for a OS. However, when so many small but variable aspects/functions interlock with each other, it’s really overwhelming. As a result, instead of working out a clever algorithm,the main problem is to manage the complexity.

  • So, when I accomplish this module, it’s very necessary to review my code and summarize my drawbacks.

  • Firstly, I prefer to the bottom-up design, which can be a good solution for a small project, but is a poor way to attack complicated project.

    • The function I write is aimed at the particular request, instead of the general one. For example, I write a function, drawTable, to generate a html table tags presenting the search result. And I write the table-header name directly in my code.

      So, next time, when I need to generate a table to present other infos, it’s impossible to reuse this function. 

- Another obvious problem of bottom-up design is that, along with the deep development of my project, I have to revise the structure and to write more abstract functions or classes. Owning to these changes, I suffer from some boring, meaningless, and repeating work.
  • What’s more, when the above two problems combine together, situation gets worse. On the one hand, I have to write new and more abstract functions to respond to the variable structure, on the other hand, barely can I reuse the existing functions. As a result, the amount of code is growing at an astonishing speed and I can hardly manage its complexity. When it’s near the end of development, I can’t even locate a function.

  • I think there are some ways to deal with these problems.

    • I should design the project in a higher level, which, however, needs a lot of practice and experience.

    • I should write codes in an extensible, modular and loosely coupled way. This method can properly counteract the attack of complexity.

  • Also, I’m seeking some other solutions by reading some classic books, such as Code Complete. I do hope I can overcome this problem.
