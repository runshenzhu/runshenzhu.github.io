basic data structure for process/thread

Posted on 2015-04-27   |  

  • When having an interview, there are some very very common questions that interviewers love to ask, including what’s the difference between stack section and heap section? or what’s the difference between thread and process?

  • For the latter one, I would alway give a very very common answer, that I learned from the book, in Linux, thread is just a lightweight process, it has its own stack and registers, such as eax, ebx, eip and so on, and shares the heap section. This answer may be right, (I hope so =。=). But there is alway one thing that I didn’t figure out.

  • How can a thread have it own registers? Since there is only one eip/eax/esp register in each cpu. What’s the meaning of having its own register?

  • Then I got a chance to implement a OS from mit 6.828. (and it’s my github repo 6.828-MIT-OS). Finally, I got that instead of having its own registers, every thread, and process maintains a struct, that represent its registers.

  • When the OS does a schedule, old thread/process saves (or we can say push) the value of its registers into this struct, and the new thread/process pops the value of this struct to registers.

  • By doing so, every thread and process seems to have its own registers.
